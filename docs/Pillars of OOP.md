# Pillars of OOP

## What are the core principles of OOP?

- **Encapsulation** — hiding implementation details; exposing only what is necessary.
- **Inheritance** — creating a new entity based on an existing one.
- **Polymorphism** — the ability for the same entity to take different forms.
- **Abstraction** — defining a set of common characteristics while ignoring irrelevant details.
- **Message passing** — the form of communication and interaction between entities.
- **Reusability** — all of the above work together to enable code reuse.

> This order is intentional: each principle builds on the ones before it.
